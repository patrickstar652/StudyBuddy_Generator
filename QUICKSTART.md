# 🚀 Study Buddy 快速入門

## 30 秒快速啟動

```powershell
# 1. 進入專案目錄
cd study_buddy

# 2. 執行啟動腳本
.\start.ps1

# 3. 等待瀏覽器自動打開
# 完成！🎉
```

## 📤 上傳你的第一份文件

```
1. 點擊上傳區域或拖放文件
   支援格式: PDF, DOCX, TXT

2. 選擇文件後點擊「開始處理」
   
3. 系統自動處理（約 3-10 秒）:
   ✓ 提取文字
   ✓ 智慧切片 (1000 tokens/區塊)
   ✓ 生成向量嵌入 (384 維)
   ✓ 建立搜索索引
   
4. 處理完成！自動跳轉到文件頁面
```

## 🎯 使用學習工具

### 生成隨堂考
```
文件頁面 → 點擊「生成隨堂考」
↓
設定題目數量 (5-10 題)
↓
選擇題型 (選擇題/簡答題/混合)
↓
點擊「生成測驗」
↓
開始答題！
```

### 生成閃卡
```
文件頁面 → 點擊「生成閃卡」
↓
設定閃卡數量 (5-20 張)
↓
點擊「生成閃卡」
↓
點擊卡片翻轉
使用 ← → 鍵切換
空白鍵翻轉
```

### 生成摘要
```
文件頁面 → 點擊「畫重點 TL;DR」
↓
設定摘要要點數 (3-10 點)
↓
點擊「生成摘要」
↓
查看核心重點和關鍵詞
```

### 智慧問答
```
文件頁面 → 找到「向文件提問」區域
↓
輸入問題，例如：
  "這份文件的主要觀點是什麼？"
  "什麼是機器學習？"
↓
點擊「發送」
↓
AI 基於文件內容回答，並顯示來源
```

## 💡 使用技巧

### 1. 文件準備
```
✅ 最佳: 10-50 頁的 PDF 或 DOCX
✅ 可以: 最多 100 頁
⚠️  太大: 建議分章節上傳
```

### 2. 問題技巧
```
✅ 好問題:
   "這份文件討論了哪些機器學習算法？"
   "監督式學習和非監督式學習的差異是什麼？"

❌ 不好的問題:
   "是"（太簡短）
   "文件裡說什麼"（太籠統）
```

### 3. 測驗設定
```
首次測驗: 5 題混合題型
深度測試: 10 題選擇題
快速複習: 5 題簡答題
```

### 4. 閃卡學習
```
第一輪: 順序學習（熟悉內容）
第二輪: 打亂順序（測試記憶）
第三輪: 只複習不熟的卡片
```

## 🔧 常見問題

### Q: 上傳失敗怎麼辦？
```
1. 檢查文件格式 (必須是 PDF, DOCX, TXT)
2. 檢查文件大小 (建議 < 50MB)
3. 確認文件沒有密碼保護
4. 嘗試轉換為 PDF 後重新上傳
```

### Q: 生成速度慢？
```
1. 檢查網路連線
2. 確認 Groq API Key 有效
3. 減少生成數量 (題目/閃卡)
4. 檔案太大可分段上傳
```

### Q: 回答不準確？
```
1. 確保問題明確具體
2. 嘗試換個方式提問
3. 檢查文件是否包含相關內容
4. 可能需要更詳細的文件
```

### Q: 如何刪除文件？
```
首頁 → 找到文件卡片 → 點擊垃圾桶圖示
（鼠標懸停在卡片上會顯示）
```

## 📚 範例學習流程

### 準備期末考試
```
Day 1: 上傳所有課程講義
Day 2: 為每份文件生成摘要，整理重點
Day 3: 生成閃卡，複習關鍵概念
Day 4: 生成測驗，自我測試
Day 5: 使用問答功能，解答疑惑
```

### 閱讀研究論文
```
Step 1: 上傳論文 PDF
Step 2: 生成 TL;DR，快速了解核心內容
Step 3: 針對不懂的部分提問
Step 4: 生成閃卡記錄重要術語
Step 5: 生成測驗確保理解
```

### 複習考試
```
1. 上傳考試範圍的材料
2. 生成綜合測驗
3. 完成測驗，檢討錯誤
4. 對錯誤的部分提問深入理解
5. 生成閃卡強化記憶
6. 定期複習閃卡
```

## 🎯 最佳實踐

### ✅ DO
- 上傳清晰、結構化的文件
